# 🎣 Fishing Game (MVP)

## 📌 Общая идея
Игра про рыбалку от первого лица:
- спокойная ловля
- поклёвка по таймингу
- вываживание с натяжением

---

# 🧱 Текущая игровая механика

## Состояния
idle → waiting → bite → reeling → reset

### idle
- игрок ещё не закинул

### waiting
- поплавок в воде
- ожидание поклёвки

### bite
- короткое окно реакции (~1–2 сек)
- поплавок дёргается

### reeling
- поплавка нет
- появляется леска
- идёт вываживание
- работает натяжение

---

# 🎮 Управление

## Мышь
- Клик → заброс / подсечка
- Зажал → тянешь
- Отпустил → ослабил

## Клавиатура
- Space → то же самое

## 7. Модель расширения (наживки, усилители, модификаторы)

Проект изначально строится с возможностью расширения игровой логики
без переписывания существующего кода.

Расширение происходит через:
- данные (data)
- модификаторы (modifiers)
- сессию ловли (CatchSession)

---

### 7.1 Общий принцип

Любая новая механика (наживка, усилитель, эффект) НЕ внедряется напрямую в экран.

Вместо этого:
1. Описывается в data
2. Преобразуется в модификаторы
3. Применяется к CatchSession

---

### 7.2 Слой данных (data)

В папке data хранятся только описания сущностей:

- fish
- baits
- boosters (усилители)
- locations

Пример структуры наживки:

```js
{
  id: "bait_worm",
  name: "Червь",
  modifiers: {
    biteChance: 10,
    preferredFish: ["crucian"]
  }
  Пример усилителя:

{
  id: "booster_luck",
  name: "Удача рыбака",
  modifiers: {
    rareChance: 5
  },
  duration: 3
}

Важно:

data содержит только данные
data не содержит логики
7.3 Слой модификаторов (services/game)

Модификаторы — это промежуточный слой между данными и механикой.

Задачи:

взять данные (bait, booster, location)
преобразовать в числовые эффекты
объединить их

Пример:

biteChance =
  fish.baseChance
  + bait.biteBonus
  + booster.biteBonus
  + location.biteModifier

Рекомендуется отдельный модуль:

services/game/modifiers.js

Функции:

getBaitModifiers()
getBoosterModifiers()
mergeModifiers()
7.4 CatchSession как точка применения

CatchSession хранит не все данные,
а итог применения всех эффектов.

Пример:

{
  baitId: "bait_worm",
  boosterIds: ["booster_luck"],

  modifiers: {
    biteChanceBonus: 10,
    rareChanceBonus: 5,
    tensionControlBonus: 0
  }
}

Важно:

в сессии нет логики расчёта
в сессии уже готовый результат
7.5 Поток данных

Последовательность работы:

Игрок выбирает:
наживку
усилители
При создании CatchSession:
берутся данные из data
рассчитываются модификаторы
формируется итоговый modifiers
Вся механика ловли работает уже с modifiers

## 8. CatchSession (ядро механики ловли)

В проекте реализована сущность CatchSession — модель одной попытки ловли.

CatchSession — это runtime-объект, который описывает:
- текущее состояние ловли
- фазу процесса
- натяжение
- результат попытки

CatchSession не хранится в БД.
Он существует только во время одной попытки ловли.

---

### 8.1 Назначение

CatchSession решает задачи:
- объединяет всю попытку ловли в один объект
- отделяет игровую логику от UI
- упрощает расширение (рыбы, наживки, усилители)
- предотвращает размазывание состояния по компоненту

---

### 8.2 Где находится

Файл:

src/services/game/catchSession.js


---

### 8.3 Структура (текущая версия)

```js
{
  sessionId: "catch_1",

  phase: "idle", // idle | waiting | bite | reeling | success | failed

  startedAt: number | null,
  biteAt: number | null,

  showFloat: boolean,
  isHolding: boolean,
  tension: number,

  result: null | "success" | "failed",
  failReason: null | "cancel" | "miss_bite" | "tension_high" | "tension_low"
}
8.4 Фазы ловли
idle — ожидание начала
waiting — поплавок в воде, ожидание поклёвки
bite — окно реакции (поклёвка)
reeling — вываживание
success — успешный улов (задел)
failed — неудача
8.5 Поток состояний
idle → startCast → waiting
waiting → triggerBite → bite
bite → startReeling → reeling
bite → failCatch(miss_bite) → failed
reeling → failCatch(tension_high) → failed
reeling → failCatch(tension_low) → failed
failed → reset → idle
8.6 Функции управления

CatchSession управляется через функции:

createCatchSession()
resetCatchSession()
startCast()
triggerBite()
startReeling()
setHolding()
updateSessionTension()
failCatch()
finishCatch() (задел)

Важно:

функции НЕ мутируют объект
всегда возвращают новый state
8.7 Связь с FishingScreen

FishingScreen:

хранит текущую session (useState)
вызывает функции перехода
управляет таймерами
отображает UI

FishingScreen НЕ:

рассчитывает механику
не хранит разрозненные состояния
8.8 Связь с fishingSession.js
catchSession.js → состояние и переходы
fishingSession.js → расчёты и формулы

fishingSession отвечает за:

увеличение/уменьшение натяжения
пороги (слишком высоко / низко)
тайминги поклёвки
8.9 Архитектурные правила

Обязательно:

вся попытка ловли = один объект session
все переходы через функции catchSession
UI не хранит логику фаз

Запрещено:

хранить phase отдельно от session
хранить tension отдельно от session
дублировать состояние в useState

Следующий этап расширения:

CatchSession — центральный узел игры:

data → даёт параметры
