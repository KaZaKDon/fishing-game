# 🎣 Fishing Game (MVP)

## 📌 Общая идея
Игра про рыбалку от первого лица:
- спокойная ловля
- поклёвка по таймингу
- вываживание с натяжением

---

# 🧱 Текущая игровая механика

## Состояния
idle → waiting → bite → reeling → reset

### idle
- игрок ещё не закинул

### waiting
- поплавок в воде
- ожидание поклёвки

### bite
- короткое окно реакции (~1–2 сек)
- поплавок дёргается

### reeling
- поплавка нет
- появляется леска
- идёт вываживание
- работает натяжение

---

# 🎮 Управление

## Мышь
- Клик → заброс / подсечка
- Зажал → тянешь
- Отпустил → ослабил

## Клавиатура
- Space → то же самое

---

# 📄 Описание файлов

## 🎣 FishingScreen.jsx
Главный файл игры:
- хранит состояние (state)
- управляет переходами:
  idle / waiting / bite / reeling
- обрабатывает:
  - клики
  - клавиши
  - удержание
- запускает таймеры
- подключает компоненты

---

## 🎯 TensionBar.jsx
UI компонент:
- принимает `value` (0–100)
- отображает шкалу натяжения
- визуально делит на зоны:
  - зелёная (норма)
  - жёлтая (опасно)
  - красная (обрыв)

---

## ⚙️ fishingSession.js
Логика игры (чистые функции):

- getBiteDelay()
- getReactionTime()
- increaseTension()
- decreaseTension()
- isTensionTooHigh()
- isTensionTooLow()

НЕ содержит React

---

## 🔧 random.js
Утилита:

- randomBetween(min, max)

Используется во всей игре

---

# 🔁 Поток логики

1. Игрок кликает → handleCast()
2. Через delay → состояние bite
3. Игрок реагирует:
   - вовремя → reeling
   - рано/поздно → reset
4. В reeling:
   - удержание → рост натяжения
   - отпуск → падение
5. Если:
   - слишком сильно → обрыв
   - слишком слабо → сход

---

# ❗ Важные принципы проекта

- НЕ используем сложную графику сейчас
- сначала механика → потом визуал
- избегаем лишнего UI
- минимум текста на экране
- игрок учится по поведению объектов
